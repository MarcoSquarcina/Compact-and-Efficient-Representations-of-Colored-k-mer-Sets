# USTAR Block-Aware (Extended from USTARC)

## Overview
**USTAR Block-Aware** is an advanced version of USTARC/USTAR3, a k-mer set compressor with support for colors. 

It is built upon the core concepts of [UST](https://github.com/medvedevgroup/UST) and [prophAsm](https://github.com/prophyle/prophasm) for SPSS representation (simplitigs), and it leverages node-revisiting techniques to achieve superior compression, as demonstrated by [Matchtigs](https://github.com/algbio/matchtigs).

**What's new in this version:** This extended version introduces the **Block-Aware** heuristic. By utilizing the compressed CSR matrix generated by the **Optimizer**, USTAR prioritizes merging nodes that share the same color block ID. This strategy maintains color coherence, reduces sequence fragmentation, and significantly enhances the downstream RLE color compression. This approach effectively solves the "color table explosion" bottleneck encountered when processing large-scale pangenomic and metagenomic datasets.

## Dependencies
No external library dependencies are required for compilation. However, for the full pipeline, you will need:
1. [GGCAT](https://github.com/algbio/ggcat): To compute the colored compacted de Bruijn graph (cdBG).
2. **Optimizer**: To process GGCAT's `colors.dat` file and generate the optimized mapping files (`.dat`, `.dict.bin`, `.map.bin`).

## Compilation
* `git clone <your-repository-url>`
* `cd USTAR_Block_Aware`
* `cmake . && make -j 4`

## Usage Guide (Standard)

1. **Run GGCAT:** `./GGCAT -k <kmer-size> -e -c <input.fasta>`
2. **Run the Optimizer:** (Process the `colors.dat` file to generate the optimized files, e.g., `final_bitmask.dat`).
3. **Run USTAR Block-Aware:**
   Provide the GGCAT FASTA graph and the base name of the Optimizer's output (without the `.dat` extension) using the `-m` parameter. To enable the block-aware logic and optimized binary compression, use `-x =b -e opt_rle`:

   ```bash
   ./ustar -k 31 -i <GGCAT-output.fasta> -m <path/to/optimizer/final_bitmask> -x =b -e opt_rle
